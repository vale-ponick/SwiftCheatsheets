# 🧠 Архитектура консольного менеджера задач (ValeToDo)

## 🎯 Суть
Разделение программы на:
- **Model** (`struct Task`) — данные задачи
- **Manager** (`class TaskManager`) — логика работы с задачами
- **Main** (`main.swift`) — ввод/вывод и команды пользователя

## 🔥 Ключевые элементы

### 1. Структура задачи
```swift
struct Task {
    let id = UUID()
    let title: String
    var isCompleted: Bool
}
2. Перечисление команд
swift
enum Command: String {
    case addTask = "add task"
    case showAll = "show all tasks"
    // ...
}
3. Класс Manager
add(_:) — добавление

displayAll() — показать все

showCompleted() / showUncompleted() — фильтрация

delete(at:) — удаление по индексу

toggleStatus(at:) — переключение статуса

4. Главный цикл
while true — бесконечное меню

readLine() — ввод команды

Command(rawValue:) — преобразование строки в enum

switch — вызов методов manager

✅ Почему это хорошо
Разделение ответственности (не всё в main)

Типобезопасные команды (enum вместо строк)

Лёгкость расширения (добавил case → добавил метод)

🔆 Использовать для:
Консольных приложений с меню

ToDoList, калькулятора, телефонной книги

Тренировки архитектуры перед перехода к UIKit/SwiftUI


📊 Сравнение двух версий ValeToDo

Критерий    V1 (Базовый)    V2 (Профессиональный)
Структура кода    Все функции глобальные, данные в global scope    Класс TaskManager инкапсулирует данные и логику
Команды пользователя    Строковые литералы в switch    enum Command: String + Command(rawValue:)
Безопасность ввода    Прямое сравнение строк    Типобезопасное преобразование в enum
Добавление новых команд    Нужно добавлять case и менять несколько мест    Добавить case в enum + реализовать метод в TaskManager
Работа с данными    Прямой доступ к глобальному массиву tasks    Методы класса, private(set) для контроля доступа
Повторный код    Циклы печати дублируются в show all, show completed, show uncompleted    Единый метод displayAllTasks() и фильтры через filter
Тестируемость    Сложно тестировать без запуска всей программы    Методы класса можно тестировать отдельно
Читаемость    Понятно новичку, но логика размазана    Каждый компонент отвечает за своё (SRP)
Расширяемость    Нужно переписывать существующий код    Добавляешь новые методы, не трогая старые
Глобальные переменные    ✅ есть (риск случайных изменений)    ❌ нет (данные внутри класса)
guard для раннего выхода    Используется только в добавлении    Используется во всех методах класса
Отображение статуса    Тернарный оператор в каждой печати    Тернарный оператор только в displayAllTasks (один раз)
Обработка неверного ввода    continue в нескольких местах    Единый guard + continue в главном цикле
✅ Плюсы V2 (класс + enum)
✅ Безопаснее — private(set) защищает массив от случайных изменений

✅ Чище — нет глобальных переменных

✅ Удобнее расширять — новый case в enum + новый метод в классе

✅ Легче тестировать — можно проверить TaskManager без main

✅ Меньше дублирования — displayAllTasks используется везде, где нужен список

⚠️ Минусы V2
❌ Сложнее для новичка — нужно понимать классы, enum, private(set)

❌ Больше кода — но оправдано для поддержки

🎯 Итог
V1    V2
✅ для быстрого прототипа    ✅ для реального проекта
✅ для обучения    ✅ для портфолио
❌ плохо расширяется    ✅ легко расширять
❌ глобальные переменные    ✅ инкапсуляция
V2 — это профессиональная версия V1 с теми же возможностями, но чище, безопаснее и удобнее для роста.
